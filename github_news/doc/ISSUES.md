# 问题清单 & 功能校验

> 用于记录现有实现的问题，以及后续功能迭代时的验收依据。
> 每个问题解决后在对应行打 ✅ 标记。

---

## 一、数据准确性

| # | 问题 | 现状 | 预期 | 状态 |
|---|------|------|------|------|
| D1 | Trending 数据源错误 | GitHub Search API 按星标总量排序 | 真实 Trending 榜（日/周/月） | ⬜ |
| D2 | 每日"热门"定义错误 | 星标绝对值最高 | 今日新增星标最多（增量） | ⬜ |
| D3 | 每周数据无历史积累 | 当天数据的聚合 | 7 天各自数据的汇总对比 | ⬜ |
| D4 | 项目解读质量低 | 关键词模板匹配（十几个固定模板） | 真实 AI 分析或人工整理 | ⬜ |
| D5 | 本周洞察硬编码 | 4 段写死的固定文字 | 根据实际数据动态生成 | ⬜ |

---

## 二、功能缺失

| # | 问题 | 说明 | 状态 |
|---|------|------|------|
| F1 | 无真实 Trending 语言筛选 | 官方 Trending 支持按语言过滤，现有 Search 查询无法精确复现 | ⬜ |
| F2 | 无时间维度区分 | Trending 分今日 / 本周 / 本月，现在只有"近期"一个含糊维度 | ⬜ |
| F3 | 无历史记录 | 关闭页面数据全丢，无法回看昨天、上周的榜单 | ⬜ |
| F4 | 无今日新增星标展示 | 卡片上看不到"今天涨了多少星"这个核心指标 | ⬜ |

---

## 三、技术局限

| # | 问题 | 说明 | 状态 |
|---|------|------|------|
| T1 | API 频率限制 | 未认证 60 次/小时，多人访问可能直接 429 报错 | ⬜ |
| T2 | 无自动更新机制 | 仅在页面刷新或手动触发时更新，无定时任务 | ⬜ |
| T3 | fallback 静态数据过时 | 备用数据写死于特定时间点，长期不维护会失真 | ⬜ |
| T4 | 每周数据非真实累计 | 完全派生自当天数据，不是 7 天的真实汇总 | ⬜ |
